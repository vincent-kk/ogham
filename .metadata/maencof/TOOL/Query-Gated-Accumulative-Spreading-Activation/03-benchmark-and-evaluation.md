# 벤치마크 및 평가 체계

## 구성 가능성 검토 결론

**가능하다.** 단, 두 가지 사실을 전제로 설계해야 한다.

1. **외부 벤치마크는 이식 불가.** 개인 지식 볼트(위키링크+폴더+태그, ≤5k 노드) 대상의 통제된 검색 품질 평가는 문헌에 존재하지 않는다(2024–2026 조사 결과). MuSiQue류 멀티홉 QA 수치는 LLM 추출 triple 그래프 전제라 maencof 위상과 다르다. 따라서 **자체 골든셋이 유일한 신뢰 가능한 척도**다.
2. **실볼트는 커밋 불가.** 운영 vault는 개인 데이터이므로 저장소에 들어갈 수 없다. 커밋 가능한 평가는 **실측 통계를 재현한 결정적 합성 픽스처** 위에서 수행하고, 실볼트 평가는 로컬 전용 러너로 분리한다.

## 2-트랙 구조

| 트랙 | 데이터 | 위치 | 실행 |
| --- | --- | --- | --- |
| T1 품질 (커밋) | 결정적 합성 vault + 골든 쿼리셋 | `src/__tests__/eval/` | vitest — CI 회귀 게이트 |
| T2 지연 (커밋) | 100/500/1000 합성 그래프 | `src/__tests__/bench/` (기존) | vitest bench — 상대 비교 |
| T3 실볼트 (로컬) | 사용자 vault 스냅샷 | 미커밋, env 지정 경로 | 수동 리포트(후속) |

## T1: 합성 픽스처 설계

픽스처는 tirnanog 실측 위상 특성을 축소 재현한다. **시드 고정 결정적 생성**(랜덤 없음, 인덱스 기반 규칙 생성)으로 실행 간 완전 동일성을 보장한다.

재현할 위상 특성:

- **레이어 분포**: L1 소수(2–3) / L2·L3 다수 / L4 활동 노드 / L5 buffer — 실측 subLayer 분포 반영
- **허브 클리크**: 20+ 노드 단일 폴더 1개(SIBLING 클리크) — 차수 정규화 효과 검증용
- **wikilink 고아율 ≥ 50%**: LINK는 소수 노드에 집중 — 실측 55% 반영
- **허브 태그**: 다수 노드가 공유하는 태그 1–2개(DOMAIN 엣지 폭발) — 시드 캡/specificity 검증용
- **다경로 수렴 구조**: 두 시드에서 각각 2홉으로 도달하는 목표 노드 — max vs sum의 판별 사례
- **어휘 중첩 사슬**: 쿼리 토큰과 제목/태그가 겹치는 경로 vs 겹치지 않는 경로 — 게이트 판별 사례

## 골든 쿼리셋

각 항목은 `(seeds, 기대 문서의 graded relevance)` 쌍이다. 등급은 3단계(2=핵심, 1=관련, 0=무관)로 하고, 쿼리 유형을 계층화한다:

1. **단일 정확 시드** (title-exact) — 국소 이웃 정밀도
2. **다토큰 수렴 쿼리** — 두 개념의 교차점 문서를 찾는가 (G1 판별)
3. **허브 태그 쿼리** — 클리크 도배 없이 유의미 결과를 내는가 (G3 판별)
4. **경로(폴더) 시드** — 폴더 맥락 확장
5. **간접 연상 쿼리** — 시드에서 2–3홉의 어휘 비중첩 문서 (γ_floor 검증)

초기 15–20쿼리로 시작해 회귀 발견 시마다 사례를 추가한다(사례 추가가 곧 지표 상향의 동력).

## 지표 정의

| 지표 | 정의 | 용도 |
| --- | --- | --- |
| **nDCG@10** | `DCG@10 / IDCG@10`, `DCG = Σ (2^rel−1)/log₂(i+1)` | 주 지표 — 등급·순위 민감 |
| **Recall@10** | 상위 10 내 rel≥1 문서 비율 | 놓침 감지 |
| **MRR** | 첫 rel=2 문서 역순위 평균 | 최상위 정밀도 |
| P95 지연 | T2 bench | 100ms 예산 감시 |

집계는 쿼리별 지표의 산술 평균(macro average)으로 한다.

## Ratchet 운영 규칙 ("점차 올리기"의 메커니즘)

`src/__tests__/eval/baseline.json`에 엔진별 지표 스냅샷을 커밋한다.

```json
{ "engine": "qga", "ndcg10": 0.xx, "recall10": 0.xx, "mrr": 0.xx, "queries": N }
```

1. **회귀 게이트**: vitest 테스트가 현재 지표 ≥ `baseline − ε`(ε=0.005, 부동소수 여유)을 단언한다. 미달 시 CI 실패.
2. **상향(ratchet)**: 지표를 개선한 변경은 같은 PR에서 `MAENCOF_EVAL_UPDATE_BASELINE=1 yarn test:run` 으로 baseline을 재기록해 커밋한다. baseline은 내려갈 수 없고 올라가기만 한다.
3. **케이스 추가 규칙**: 골든셋에 쿼리를 추가하면 baseline을 같은 커밋에서 재기록한다(지표의 분모가 바뀌므로). 케이스 추가로 인한 수치 하락은 회귀가 아니다.
4. **ablation**: 게이트(γ_floor)·정규화(`/deg` vs `/√deg`)·융합 리스트 구성은 eval 러너의 파라미터 오버라이드로 on/off 비교한다.

이 구조로 "샘플을 만들어 벤치마크 → 지표를 점차 상향"의 반복 루프가 성립한다: 실사용에서 아쉬운 검색을 발견 → 골든셋에 사례 추가 → 파라미터/알고리즘 개선 → ratchet 상향 커밋.

## 한계와 완화

- **합성 픽스처의 대표성**: 합성 위상이 실볼트의 어휘 분포를 완전히 재현하지는 못한다. → T3 로컬 러너(실볼트 스냅샷 + 사용자가 직접 매긴 골든셋)를 후속 도입해 교차 검증한다.
- **골든셋 저자 편향**: 기대 문서를 설계자가 정하므로 순환 논증 위험이 있다. → 쿼리 유형을 판별 사례 중심으로 구성하고(위 5유형), 실사용 불만 사례를 우선 추가한다.
- **지표 게이밍**: 특정 골든셋 과적합. → 케이스 추가 규칙(3)과 ablation 보고를 관행화한다.
