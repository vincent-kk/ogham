# NPDP 핵심 고민 — 개발 완료 코드의 안정성 보장

## 문제 정의

NPDP Phase 3~4 영역. AI가 생성하고 개발 완료된 코드의 **안정성을 어떻게 보장**할 것인가. ogham의 filid가 "코드 문서화 + AI 기반 코드 안정장치"로 정의되어 있지만 구체 메커니즘이 부재.

## PL로서 답해야 할 핵심 질문

### 안정성의 정의
- "안정성"이란 구체적으로 무엇인가? (테스트 커버리지? 회귀 방지? 타입 안전성? 런타임 모니터링?)
- 안정성의 정량적 기준은?
- 안정성 수준을 계층화할 수 있는가? (Core는 높게, 실험적 코드는 낮게)

### AI가 생성한 코드를 AI가 검증하는 구조
- 이 구조의 신뢰성 문제를 어떻게 해결하는가?
- 생성 AI와 검증 AI를 분리하는 전략? (다른 모델? 다른 프롬프트? 다른 관점?)
- 인간 검증은 어느 지점에서 개입하는가?

### 사이드이펙트 방지
- 기능 추가 시 **기존 코드를 건드리지 않는 보장** 전략은?
- filid의 "안정장치" 역할을 구체적으로 어떻게 구현하는가?
- 변경 영향도 분석(impact analysis)을 자동화할 수 있는가?
