# NPDP 핵심 고민 — AI 코드 발산 방지 & 철학 주입

## 문제 정의

NPDP Phase 2~3 영역. AI가 코드를 생성할 때 설계자의 의도와 철학에서 벗어나는 **발산(divergence)**을 방지하고, "나의 철학"을 코드에 일관되게 반영하는 방법을 설계해야 한다.

현재 AX-Native PRD Template 2단계에 "AI 가드레일/제약조건 설정"이 언급만 되어 있고 구체 메커니즘이 부재.

## PL로서 답해야 할 핵심 질문

### "나의 철학"의 정의
- 구체적으로 무엇인가? (코딩 컨벤션? 아키텍처 원칙? 설계 패턴? 네이밍? 모두?)
- 이것을 **문서화 가능한 형태**로 어떻게 체계화하는가?
- 철학이 진화할 때 업데이트 프로세스는?

### AI에게 주입하는 방법
- 어떤 형태로 전달하나? (CLAUDE.md 룰셋? 코드 템플릿? few-shot 예제? 타입 시스템?)
- ogham이 이 역할을 담당하는가? ogham의 어떤 기능으로?
- 프로젝트별로 다른 철학을 적용할 수 있는 구조인가?

### 발산 감지 시점
- 생성 시점에 제약 (사전 방지) vs 생성 후 검증 (사후 탐지) — 어느 전략?
- 발산을 감지하는 기준/메트릭은?
- 발산이 감지되면 어떤 액션을 취하는가? (자동 수정? 인간 에스컬레이션?)

## 현재 상태

- [ ] 주입 메커니즘 설계 (ogham 연계)
- [ ] 발산 감지 기준 및 대응 프로세스 정의
