# imbas Phase 1 — 정합성 검증 워크플로우 (v1)

## 목적

비정형 기획문서(복수 가능)를 정형화한 뒤, Phase 2(이슈 분할) 진입 전에 **논리적 결함을 사전 차단**하는 게이트.

---

## 흐름

```
복수 비정형 문서
  → [imbas:classify] 합치기 + 정형화 (내부 로직)
  → 단일 정형 문서
  → [imbas:validate] 정합성 판정
  → 검증 리포트 (마크다운)
  → 이슈 있음 → 🧑 블로킹 (인간이 원본 수정)
  → 이슈 없음 → Phase 2 진입
```

### classify 단계

- 복수 문서를 하나의 정형 문서로 합침
- 내부 로직으로 사용자에게 투명한 동작
- 원본 불변 원칙 유지 — 원본은 읽기 전용, 정형 문서는 별도 생성

### validate 단계

- 합쳐진 단일 정형 문서를 대상으로 검증
- 복수 문서 간 교차 검증이 아님 (classify에서 이미 합침)

---

## 검증 대상 4종

| 유형 | 정의 | 감지 방법 (초안) | 예시 |
|------|------|-----------------|------|
| **모순** | 같은 대상에 대해 양립 불가능한 요구 | 문서 내 교차 비교 | "로그인 없이 접근 가능" vs "인증 필수" |
| **이격** | 상위-하위 간 논리적 단절 | 상위 목표→하위 스펙 트레이싱 | 상위에서 "실시간"이라 했는데 하위에 배치 처리 스펙 |
| **누락** | 논리적으로 있어야 하지만 없는 스펙 | 입력-출력 체인 검사 | 에러 케이스 미정의, 경계값 미정의 |
| **논리적 불능** | 기술/논리적으로 원리상 불가능한 요구 | LLM 판단 | "오프라인에서 실시간 동기화" |

### Phase 1에서 하지 않는 것

| 유형 | 이유 | 처리 시점 |
|------|------|-----------|
| **모호** | 1차 분리 과정에서 자연스럽게 감지 | Phase 2 |
| **코드상 불능** | 코드 참조 필요, 문제 공간 범위 밖 | 별도 단계 (미설계) |

---

## 리포트 형태

- **기본**: 마크다운 리포트 — 유형별 분류, 근거 인용, 위치 표시
- **선택**: 요청 시 Jira 이슈 코멘트로 발행 (별도 액션)

### 리포트 구조 (초안)

```markdown
# imbas Validation Report
source: [정형 문서 식별자]
date: 2026-04-04

## 🔴 모순 (N건)
### V-C01: [제목]
- 위치 A: "..." (섹션 X)
- 위치 B: "..." (섹션 Y)  
- 판정: 양립 불가 — [근거]

## 🟠 이격 (N건)
### V-D01: [제목]
- 상위: "..." (섹션 X)
- 하위: "..." (섹션 Y)
- 판정: 논리적 단절 — [근거]

## 🟡 누락 (N건)
### V-M01: [제목]
- 맥락: "..." (섹션 X)
- 기대 스펙: [논리적으로 있어야 할 내용]
- 판정: 미정의 — [근거]

## 🔴 논리적 불능 (N건)
### V-I01: [제목]
- 위치: "..." (섹션 X)
- 판정: 원리적 불가 — [근거]

## ✅ 검증 통과 항목 요약
[정상으로 판정된 주요 영역 목록]
```

---

## 블로킹 정책

- Phase 1 검증 이슈 발견 → **Phase 2 진입 블로킹**
- 인간이 원본 수정 후 재검증 (imbas:validate 재실행)
- Phase 2 이후 발견된 검증 오류 → **해당 단계에서 블로킹**

---

## 실행 환경

- Claude Code 플러그인 (imbas) 내 스킬
- LLM: Haiku / Sonnet / Opus 중 선택
- 세부 감지 로직은 추후 구체화 (목적과 형태만 v1에서 확정)
