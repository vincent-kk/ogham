# seiri — 설계 문서

> **seiri** (웨일스어 `saer`의 복수형, "장인들") — 코드 작성·품질·개발방법론을 담당하는 Claude Code 플러그인.

`.metadata/seiri/`는 seiri의 **한국어 설계 정본(SSoT)**입니다. 구현체는 `plugins/seiri/`이며, 배포되는 규칙 본문은 영문입니다.

---

## 무엇인가

seiri는 **에이전트가 파악하기 좋은 코드 구조를 추종하는** 규칙 세트입니다. 특정 아키텍처에 종속되지 않고 어떤 저장소에서도 단독으로 동작합니다.

|             |                                                                                                     |
| ----------- | --------------------------------------------------------------------------------------------------- |
| **제1원칙** | "에이전트가 파악하기 좋은 코드 구조를 추종한다"                                                     |
| **역할**    | 코드 작성 · 코드 품질 · 개발방법론                                                                  |
| **전제**    | FCA 비종속 — 단독 동작 가능한 범용                                                                  |
| **형태**    | `.claude/rules/`에 배포되는 규칙 문서 + 스킬 11종(작업 주기 전 구간 — 02-ARCHITECTURE §3 스킬 지도) |

## 왜 필요한가

기존 두 선례를 정밀 분석한 결과, **제1원칙을 규칙 층위로 세운 도구가 없었습니다.**

- **superpowers** — TDD 방법론 주입기. 얇고(부트스트랩 62줄) 잘 만들어졌으나, 모든 저장소에 같은 방법론과 같은 강도를 전제합니다.
- **oh-my-claudecode** — 에이전트 오케스트레이션 레이어. 강력하나 대형이며(md 47k줄 / ts 356k줄), 모든 저장소에 같은 통제를 전제합니다.

두 저장소를 통틀어 "에이전트가 읽기 좋은 코드"를 말하는 대목은 **약 25줄**이었고, 그마저 다른 스킬 본문에 묻혀 독립 규칙으로 승격되지 않았습니다. seiri는 그 자리를 채우고, 나아가 superpowers가 덮던 **작업 주기의 절차**(계획→수행→구현→검증→통합)까지 seiri 형태로 흡수해 **완전 대체**를 지향합니다(2026-07-23 결정 — 강압 부트스트랩·오케스트레이션은 이식하지 않음).

## 설계 사상

> **우리는 개별 저장소의 *값*을 모른다. 그러나 모든 코드베이스가 공유하는 *형식*은 안다.**

| 모르는 것 — **값** | 아는 것 — **형식**                           |
| ------------------ | -------------------------------------------- |
| 검증 명령, 임계치  | 검증 수단이 존재하고 실행 가능하다           |
| 언어·프레임워크    | 코드는 파일에 담기고 이름·크기·경로를 갖는다 |
| 팀 관례            | 기존 스타일이 이미 존재한다                  |
| 모듈 경계 위치     | 공개/비공개 구분이 존재한다                  |

이 분리가 두 기둥을 동시에 세웁니다 — **값을 모르므로 오라클·임계를 소유하지 않고, 형식을 알므로 보편 규칙을 팔 자격이 있습니다.**

## filid와의 관계

두 플러그인은 **위계로 분담**합니다. 이름 층위에서 이미 표현되어 있습니다.

|      | filid                               | seiri                             |
| ---- | ----------------------------------- | --------------------------------- |
| 어원 | 고대 아일랜드의 **시인·학자 계급**  | 웨일스의 **장인 계급**            |
| 담당 | 아키텍처·문서·리뷰 — 지식의 질서    | 코드 작성·품질 — 만드는 손의 규율 |
| 규칙 | **임계**(숫자와 그 의미)            | **원칙**(방향)                    |
| 예   | "spec 파일당 15케이스, 3+12의 의미" | "테스트의 무한 축적을 금지한다"   |

seiri는 방향만 제시하고, 구체적 임계값은 저장소나 filid가 선언합니다(하방 개방).
