cennad / settings

Routing, intervention, and provider defaults for codex (OpenAI), antigravity (Google), and Anthropic (claude) dispatch.

Providers

Set each provider's weight, click a provider to enable or disable it, or the gear to reveal advanced flags. Weights are normalized across enabled providers.

Install: npm install -g @openai/codex — reload after installing.

default tier

Tier applied when a dispatch omits an explicit tier. Higher is stronger but more rate-limit/budget-prone — mid suits almost all work.

recency factor

auto recommends source-checking for time-sensitive facts. strict requires search-verification with citations or an "unverifiable" fallback. codex defaults to off to prioritize cache reuse.

sandbox

Disabled — yolo bypasses the sandbox.

⚠ Full filesystem access — only use in a trusted environment.

per-tier model & effort

Map each tier to a model and reasoning effort. Models come from codex debug models, and effort options adapt to the model — codex rejects an effort the model does not advertise instead of downgrading it.

Install: curl -fsSL https://antigravity.google/cli/install.sh | bash — reload after installing, then sign in with agy.

default tier

Tier applied when a dispatch omits an explicit tier. Higher is stronger but more rate-limit/budget-prone — mid suits almost all work.

recency factor

auto recommends source-checking for time-sensitive facts. strict requires search-verification with citations or an "unverifiable" fallback. Antigravity has web-tool access by default, so auto is recommended for recency.

Antigravity has no container sandbox. Permission Allow/Deny/Ask lists are managed inside agy via /permissions; cennad follows your agy settings.json. Leaving "skip permissions" off keeps your Deny list as the safety net.

per-tier model mapping

Antigravity serves multiple model families. Map each tier to a model. Options come from agy models (run agy and sign in to populate them).

Install: npm install -g @anthropic-ai/claude-code — reload after installing.

default tier

Tier applied when a dispatch omits an explicit tier. Each tier maps to a model + effort below.

recency factor

auto recommends source-checking for time-sensitive facts. strict requires search-verification with citations or an "unverifiable" fallback.

permission mode
⚠ bypassPermissions skips all approvals — only use in a trusted environment.

claude-code has no sandbox; isolation is permission-based. cennad always adds --strict-mcp-config and --safe-mode so the child never inherits this session's MCP servers, hooks, or CLAUDE.md.

per-tier model & effort

Map each tier to a model and reasoning effort. Effort options adapt to the model (haiku has none; sonnet skips xhigh). Over-cap requests are silently downgraded by claude-code.

Artifacts

Mirror successful responses to disk as markdown files with YAML front-matter. Useful for PR attachment and session review. artifact_path is added to the envelope on success.

location

project → <cwd>/.cennad/artifacts/. user → <active cennad home>/artifacts/<projectHash>/. The active home defaults to ~/.claude/plugins/cennad/ and follows CENNAD_CONFIG_PATH when set.
