# seiri

> 웨일스어 `saer`("장인")의 복수형. 만드는 손의 규율을 다루는 규칙.

**에이전트가 파악하기 좋은 코드 구조를 추종한다** — 이 하나를 제1원칙으로
삼는 Claude Code 플러그인입니다. 코드 작성·리뷰 규율·개발 방법론에 관한
소수의 규칙을 담고, 그중 고른 것만 저장소에 배포합니다.

English: [README.md](./README.md)

## 무엇을 하는가

선택한 규칙은 `.claude/rules/seiri_*.md` 로 쓰이고, 하니스가 세션 시작마다
자동으로 읽어 들입니다. seiri 는 규칙 본문을 주입하지 않습니다 — 같은
컨텍스트를 두 번 쓰게 되기 때문입니다.

대신 주입하는 것은 파일이 스스로 말할 수 없는 것뿐입니다. 지금 어떤 규칙이
활성인지, 개입 강도가 어디 있는지, 배포된 규칙 중 원본에서 편집된 것이
있는지.

## 설치

```
/plugin install seiri
/seiri:setup
```

`/seiri:setup` 은 사용 가능한 규칙 목록을 담은 로컬 설정 페이지를 엽니다.
저장 전까지 아무것도 쓰이지 않으며, **무엇이 바뀌는지 diff 로 먼저**
보여줍니다. 이 파일들은 저장소에 상주하는 지시가 되므로, 들어오기 전에
사용자가 결정합니다.

## 설계

**저장소의 진실을 소유하지 않습니다.** 어떤 검증 명령을 쓰는지, 임계치가
얼마인지, "완료"가 무엇인지는 테스트·CI·`CLAUDE.md` 의 몫입니다. seiri 는
그 값의 사본을 갖지 않습니다 — 사본은 드리프트하기 때문입니다.

**모델의 판단을 소유하지 않습니다.** 차단 훅도 승인 게이트도 없습니다.
규칙은 컨텍스트이고, 무엇이 컨텍스트일 뿐이며 무엇이 실제로 강제되는지를
플러그인이 숨기지 않고 표기합니다.

**모든 규칙은 물러섭니다.** 각 규칙은 같은 우선순위 사슬로 시작합니다 —
저장소의 명시 지침 > 저장소의 기존 관례 > 이 규칙. 충돌하면 규칙이
비켜섭니다.

**모든 규칙은 선택입니다.** 필수 규칙이 없고, 명시적 선택 없이 배포되는
것도 없습니다.

## filid 와의 관계

둘은 층위로 나뉩니다. [filid](../filid) 는 아키텍처·임계·리뷰를 맡습니다 —
숫자와 그 숫자의 의미. seiri 는 방향을 맡습니다 — 숫자를 뺀 원칙이라야
특정 아키텍처를 모르는 저장소에서도 성립하기 때문입니다.

## 스킬

직접 호출하는 것:

| 스킬                | 용도                                    |
| ------------------- | --------------------------------------- |
| `/seiri:setup`      | 이 저장소가 쓸 규칙 선택                |
| `/seiri:brainstorm` | 코드를 쓰기 전에 변경의 형태를 잡기     |
| `/seiri:interview`  | 모호한 요청을 실패 가능한 기준으로 변환 |
| `/seiri:finish`     | 끝난 작업을 통합할지 결정               |

순간이 맞으면 자동으로 발동하는 것:

| 스킬                     | 용도                                  |
| ------------------------ | ------------------------------------- |
| `/seiri:write-plan`      | 다단계 작업을 리뷰 가능한 태스크로    |
| `/seiri:execute`         | 작성된 계획을 완료까지 수행           |
| `/seiri:implement`       | 변경을 테스트 우선으로 구현           |
| `/seiri:trace-structure` | 판단 전에 연결관계와 데이터 흐름 추적 |
| `/seiri:trace-cause`     | 실패를 드러난 곳이 아니라 시작점까지  |
| `/seiri:verify-done`     | 완료를 선언하기 전에 주장 검증        |
| `/seiri:request-review`  | 범위를 고정해 리뷰에 넘김             |
| `/seiri:receive-review`  | 리뷰 피드백을 코드에 반영             |
