# CLAUDE.md — @ogham/maencof-lens

Read-only vault knowledge access plugin (v0.1.0). Wraps maencof's knowledge graph tools for use in development contexts.

## Commands

```bash
yarn build          # clean → version:sync → esbuild (mcp-server + hooks)
yarn build:plugin   # esbuild bundle only (mcp-server + hooks)
yarn typecheck      # TypeScript type check (no emit)
yarn test:run       # single run (CI)
```

## Architecture

**Read-Only Wrapper**: Imports 5 handlers from `@ogham/maencof` (workspace dep) and wraps them with multi-vault routing + layer filtering.

**MCP Tools (5)**: `search`, `context`, `navigate`, `read`, `status`

**Skills (3)**: `maencof-lens-setup` (config), `maencof-lens-lookup` (search + read + summarize), `maencof-lens-context` (token-budgeted assembly)

**Agents (1)**: `researcher` — autonomous multi-tool vault exploration

**Hooks (1)**: SessionStart — config detection + skill usage guide injection

> Detailed docs: `../../.metadata/maencof-lens/` (design-spec, blueprint, phase-specs, work-plan)

## Always do

- Read vault path from config; never hardcode
- Apply layer guard (vault config ∩ tool param) on every tool call
- After hook or build changes, rebuild with `yarn build`

## Ask first

- Before adding new MCP tools beyond the 5 read-only set
- Before changing maencof handler imports

## Never do

- Write to vault filesystem
- Call mutation handlers (maencof_create, kg_build, etc.)
- Modify `bridge/` output files directly (generated by esbuild)
